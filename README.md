Конструктор тайлайна расследований инцидентов информационной безопасности (Incident Response Timeline).

Приложение помогает IR-команде вести дела по инцидентам: принимать алерты в очередь триажа, эскалировать их в дела (по одному или сразу пачкой), реконструировать граф связей между событиями атаки, размечать факты по MITRE ATT&CK, прикреплять артефакты и IOC, обсуждать события в комментариях и выгружать дело в JSON для отчётности.

## Возможности

- **Алерты**: очередь триажа с фильтрами, эскалация в дело (одиночная и массовая), привязка/отсоединение алертов от дела.
- **Дела**: статусы, критичность, гриф конфиденциальности, статус подтверждения; вкладка «Описание» — классификация, внешний тикет, хронология инцидента, причина, влияние, атрибуция; экспорт в JSON.
- **Таблица событий**: список фактов дела с типом, достоверностью и MITRE-разметкой.
- **Граф**: конструктор графа связей между событиями — узлы-факты (сетевое соединение, событие логона, операция с файлами, выполнение команды) с масштабированием и панорамированием, типизированные связи между узлами.
- **IOC и артефакты**: индикаторы компрометации, загрузка файлов-доказательств с WORM-защитой.
- **Комментарии**: обсуждение фактов внутри дела.
- **Панель администрирования**: пользователи, роли и группы, уведомления на почту и в Telegram-бота, часовой пояс, зашифрованные паролем бэкапы конфигурации и базы данных.
- **Роли и RBAC**: администратор, руководитель IR, следователь, threat hunter, наблюдатель, юрист, внешний подрядчик.
- **Темы оформления**: светлая, тёмная и «Саурон» (пасхалка).

## Стек технологий

- **Backend**: FastAPI (Python), SQLAlchemy (async), PostgreSQL, Redis, MinIO (S3-совместимое хранилище артефактов), WebSocket для совместной работы в реальном времени.
- **Frontend**: React + TypeScript, Vite, Zustand.
- **Инфраструктура**: Docker Compose, Nginx, опционально Keycloak/OIDC для SSO.

## Быстрый старт

```bash
cp .env.example .env
make dev     # dev-стек с hot-reload
# либо
make up      # production-стек
```

Frontend будет доступен на `http://localhost:3000` (dev) или через Nginx на `http://localhost` (production).
